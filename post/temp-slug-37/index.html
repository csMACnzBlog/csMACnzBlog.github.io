Gonna Build Me A Blog... Frontend

Oct 7, 2010 3 min read

So Ive been toying with the idea for a while, mulling it over. And I think I’ve finally found a way to ‘easily’ get my blog using Silverlight for a front end. After much inspiration fromJustin Angel , and a dig around his html source, as well as a good read though some SEO posts, I’m feeling pretty good about my idea. basically its this: I have a LAMP web hosting setup, using the WordPress as my blog engine. I’m really happy with my provider and with my WordPress blog and have no real intention of changing this, yet the idea of building a skin using php,html, CSS,Javascript… Not really my kind of dev, im a C based developer really. The fact that the skin on this site has been a barely modified version of one I grabbed off a skins page for over 2 years says as much. And Silverlight with its awesome xaml and C# being my tools of choice at the moment, I would really like to use that to build the UI for my site. Several factors are driving this choice for me. Interoperability, Avoidance, Time, and Migration. Basically If I do this right, I will have a personal example of Silverlight being used outside the Microsoft stack. This can only be a good promotional tool to help get more people talking Silverlight, using and choosing it for there RIA development of choice. The second is, as stated above, that I don’t want to have to do the whole normal web development stack. I’m not saying that I cant do it, just that I don’t want to. This solution is going to have its pockets of html and javascript, css and php so don’t think I’m being naive and building a site without them, that would just be crazy! (Please post comment with all crazy suggestions on how to build a site without html, would love to hear them). The other driving force is that Idon’t have the time to build an entire blog engine from scratch, and although the idea of starting small crossed my mind, i would still be building a service layer in php that Silverlight could consume, and all that other stuff a blog needs behind the scenes, as well as maintaining it all, not to mention it wouldn’t have all the nice well developed blog features such as live writer integration etc. So building on top of WordPress should be easier then building from scratch (one hopes). The last excuse for not doing this properly would have to be that I’m just not that keen on having to migrate my past postings. Silverlight 3+ has some useful features I believe will make this a bit easier: The navigation framework gives me a navigation fragment component in the browser’surl, which I can use to provide permi-links to blog posts. Silverlight 4 adds the ability to provide custom context menus, so I can make right click a more familiar experience, especially for Cut, Copy & Paste. [MORE] So lets reiterate what it is I plan to do: Build a Silverlight blog UI. Use WordPress as my underlying engine. Implement successful SEO features so i can be Googled on Bing For good measure, here are my feature requests to myself: Twitter integration Graceful degradation for non-sl users Silverlight Admin Pages the first step is going to be re skinning my blog to a basic html version that can be easily parsed and interpreted from Silverlight over the DOM bridge, which is going to be the simplest integration point to Silverlight (at least that’s my plan at this stage), and will be the basis of how I will handle Search Engine Optimisation. Stay Tuned for the next stage of this somewhat unorthodox development.
